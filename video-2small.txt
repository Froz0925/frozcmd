Froz Video recode script (video-2small.cmd)
-------------------------------------------
Пакетное сжатие видео через ffmpeg: меньше размер, минимум потерь качества.

Автоматически:
   - Поворот / масштаб - 720p/1080p по настройке
   - Частота кадров - VFR(переменный) -> CFR(постоянный) для совместимости
   - Сохранение Full Range цвета (yuvj420p)
   - Аудио - копирование или стерео Opus/AAC
   - Субтитры - копирование для MKV, вшивание (hardburn) для MP4
   - Чистка устаревших метатегов

Мин. ОС: Windows 10.

Использование:
   - Отредактируйте video-2small.ini (создается при первом запуске)
     любым текстовым редактором, например Блокнот.
   - Перетащите видеофайлы на скрипт (не папки).
   - Результаты будут в исходной папке, с суффиксом _sm, логи - в папке logs.

Масштабирование (SCALE):
   - 1 = до 720p, если высота после учёта поворота >720.
   - Пусто = до 1080p, если высота после учёта поворота >1080.
     Сохраняет FullHD для ландшафта, но не допустит, например, высоту 1920 для вертикальных видео.

Качество (CRF):
   - Меньше - лучше качество, больше файл. Рекомендуемые диапазоны по кодекам - в комментариях INI.

Контейнер (OUTPUT_EXT):
   - MKV (рекомендуется) или MP4. В MP4 добавляется флаг для быстрого старта при стриминге.
   - Язык дорожек меняется на "rus" кроме видео.
   - Удаляются устаревшие теги битрейта и размера потока, чтобы плееры не путались.

Аудио (AUDIO_ARGS):
   - По умолчанию Opus 128k (MKV) или AAC 192k (MP4).
   - При перекодировании звук сводится к стерео (2.0) для совместимости с любыми устройствами.

Поворот (ROTATION):
   - -90 / 90 / 180 - ручной поворот поверх авто-тега из MP4/MOV 
   - Кодеки *qsv не поддерживают transpose - ручная установка игнорируется.

Кодеки (CODEC) и требования:
   - Порядок по качеству/совместимости: AV1 > HEVC > H.264 (CPU/GPU-версии).
   - Для GPU-кодеков выставлены ключи максимального качества.
   - CPU-кодеки: libsvtav1 (preset 6), libx265/libx264 (preset slow)
     Медленно, но даже при этих принятых для видеоархива не самых медленных пресетах дадут качество выше GPU-версий.
   - GPU NVIDIA: требуется драйвер v570+. av1_nvenc - RTX 40xx+ (2022+).
     hevc_nvenc и h264_nvenc - GTX 9xx+ (2014+, не GT/MX). 10-bit - GTX 1050+ (2016+).
     https://developer.nvidia.com/video-encode-decode-support-matrix
   - GPU AMD: требуется драйвер AMD Adrenalin Edition v22.12.1+ (SDK 1.4.28).
     av1_amf - RX 7xxx+ (2022+) и Ryzen 7xxx+ iGPU (2022+).
     hevc_amf и h264_amf - RX 4xx+ (2016+).
   - GPU Intel: требуется Intel HD Graphics Driver + Media Feature Pack.
     av1_qsv - Arc, Core Ultra (2022+).
     hevc_qsv и h264_qsv - ix-6xxx+ (2015+). 10-bit - ix-7xxx (2017+).

Частота кадров (FPS):
   - Переменный FPS (VFR) автоматически преобразуется в постоянный (CFR)
     для совместимости с ТВ и аппаратными плеерами.
   - Чересстрочное видео (50i/60i) -> bwdif-деинтерлейс в 50p/60p (плавность без артефактов).
     Если для чересстрочного видео нужен эффект "как фильм" - можно установить параметр в 25/30.

Субтитры:
   - Обрабатывается только первая дорожка.
   - MP4: SRT/ASS вшиваются в видео (hardburn) - для отображения в мессенджерах/облаках.
     Графические (PGS) игнорируются.
   - MKV: копируются отдельной дорожкой.

Особенности:
   - Если в исходнике цвет Full Range (yuvj420p) - контейнер принудительно меняется на MKV
     и через mkvpropedit добавляется метка colour-range=1, так как MP4 не поддерживает
     надежную запись этой метки.
   - Примерное время кодирования рассчитывается по вашим коэффициентам SPEED_*,
     которые которые можно уточнить по факту кодирования на вашем железе.
   - Перед работой скрипт проверяет, поддерживает ли ваша видеокарта выбранный GPU-кодек и профиль 8/10 бит.
